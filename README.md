# 🤖 Бот курса — инструкция запуска

## Структура файлов

```
coursebot/
├── config.py        ← ВСЕ НАСТРОЙКИ ЗДЕСЬ (трогай только этот файл)
├── bot.py           ← логика бота (не трогай)
├── db.py            ← база данных (не трогай)
├── schedule.py      ← расписание (не трогай)
├── requirements.txt ← зависимости
└── Procfile         ← для Railway/Render
```

---

## ШАГ 1 — Узнай свой Telegram ID

1. Открой Telegram → найди бота **@userinfobot**
2. Напиши `/start`
3. Бот пришлёт твой числовой ID, например `123456789` — запомни

---

## ШАГ 2 — Настрой config.py

Открой **config.py** и измени:

```python
BOT_TOKEN  = "ВСТАВЬ_ТОКЕН_СЮДА"   # ← токен от @BotFather
CURATOR_ID = 123456789               # ← твой Telegram ID
```

---

## ШАГ 3 — Добавь ссылки на видео

В config.py, в массиве MODULES, замени ссылки:

```python
"video": "https://youtu.be/ССЫЛКА_МОДУЛЬ_1",
```

Работает с **любыми** ссылками: YouTube, Google Drive, Vimeo, облако.

---

## ШАГ 4 — Настрой дни открытия модулей

В config.py измени поле `"day"` у каждого модуля:

```python
{ "number": 1, "day": 0,  ... },   # сразу при старте
{ "number": 2, "day": 6,  ... },   # через 6 дней
{ "number": 3, "day": 12, ... },   # через 12 дней
...
```

---

## ШАГ 5 — Запуск

### Вариант А: Локально (для теста)

```bash
pip install -r requirements.txt
python bot.py
```

### Вариант Б: Railway (бесплатно, 24/7) — РЕКОМЕНДУЕТСЯ

1. Создай аккаунт на **railway.app**
2. Загрузи все файлы в GitHub-репозиторий
3. В Railway: **New Project → Deploy from GitHub**
4. Добавь переменные окружения (Settings → Variables):
   - `BOT_TOKEN` = твой токен
   - `CURATOR_ID` = твой Telegram ID
5. Railway сам установит зависимости и запустит бота

### Вариант В: Render (тоже бесплатно)

1. Создай аккаунт на **render.com**
2. New → **Background Worker** → Connect GitHub
3. Build Command: `pip install -r requirements.txt`
4. Start Command: `python bot.py`
5. Добавь Environment Variables: BOT_TOKEN, CURATOR_ID

## Как работает бот

| Событие | Что происходит |
|---------|----------------|
| Студент пишет `/start` | Регистрация + приветствие + Модуль 1 |
| Каждый час | Бот проверяет расписание и отправляет новые модули |
| Студент отправляет файл/фото/текст | Бот принимает как ДЗ, пересылает тебе, подтверждает студенту |
| За 24 часа до дедлайна | Автоматическое напоминание (если ДЗ не сдано) |

## Команды бота (для студентов)

| Команда | Что делает |
|---------|-----------|

## Частые вопросы

**Бот не отвечает сразу — что не так?**
Первая проверка модулей происходит через 1 час после запуска. Это нормально.

**Как поменять текст сообщений?**
Всё в config.py — разделы WELCOME_TEXT, HW_CONFIRM_TEXT и тексты в MODULES.

**Как добавить студента вручную?**
Никак — студент сам пишет `/start` боту.

**Что если студент пишет после дедлайна?**
Бот сообщает, что дедлайн истёк, и просит написать куратору напрямую.

**Где хранятся данные?**
В файле `students.json` рядом с ботом. На Railway/Render он создаётся автоматически.

**Важно:** На Railway/Render файл students.json сбрасывается при каждом деплое!
Для постоянного хранения добавь Railway Volume или используй бесплатную MongoDB Atlas.
